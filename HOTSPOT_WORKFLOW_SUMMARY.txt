CLASSIFICATION LOGIC
================================================================================

Severity Score (0-100):
  severity = (density_score * 0.4) + (trend_score * 0.3) + (resource_score * 0.3)

  Density Scoring:
    HIGH (>= 50 events/sq km) = 40 points
    MEDIUM (15-50 events/sq km) = 20 points
    LOW (< 15 events/sq km) = 5 points

  Trend Scoring:
    GROWING (> +15% change) = 30 points
    STABLE (-15% to +15%) = 15 points
    SHRINKING (< -15%) = 5 points

  Resource Efficiency Scoring:
    UNDER_RESOURCED (efficiency < 0.5) = 30 points
    OPTIMIZED (efficiency 0.5-0.7) = 15 points
    OVER_RESOURCED (efficiency > 0.7) = 5 points

Resource Allocation Decision Rules:
  IF density > 50 events/sq km OR recent_spike OR (GROWING AND efficiency < 0.7)
     THEN UNDER_RESOURCED
  ELSE IF density < 15 AND SHRINKING AND efficiency > 0.7
     THEN OVER_RESOURCED
  ELSE
     OPTIMIZED

Priority Ranking:
  - All hotspots ranked by severity_score (descending)
  - Highest severity = priority rank 1
  - Can filter by severity_score >= threshold (e.g., >= 50)


USAGE EXAMPLE
================================================================================

from socrata_toolkit.analysis.hotspot_workflow import GeographicHotspotWorkflow

# Initialize workflow
workflow = GeographicHotspotWorkflow(
    violations_fourfour="dntt-gqwq",
    complaints_fourfour="erm2-nwe9",
    inspections_fourfour="p7ve-f997",
)

# Run analysis
result = workflow.run(borough_filter="MN", sample_size=5000)

# Access results
print(f"Hotspots: {result['summary']['total_hotspots']}")
print(f"High-severity: {result['summary']['high_severity_count']}")

for hotspot in result['hotspots'][:3]:
    print(f"  {hotspot['hotspot_id']}: "
          f"severity={hotspot['severity_score']:.0f}, "
          f"trend={hotspot['trend']}, "
          f"allocation={hotspot['resource_allocation']}")

# Save map & report
with open("map.html", "w") as f:
    f.write(result['map_html'])

import json
with open("report.json", "w") as f:
    json.dump(result['final_report'], f, indent=2)


KEY FEATURES

INTEGRATION POINTS
================================================================================

Socrata Data Fetching:
  - Respects SOCRATA_APP_TOKEN for full-corpus access
  - Applies WHERE filters for geometry + borough constraints
  - Handles schema detection for location/geometry columns

Spatial Analysis:
  - DBSCAN from sklearn.cluster
  - Gaussian KDE from scipy.stats
  - Converts lat/lon to standardized coordinates

Claude API:
  - Model: claude-haiku-4-5-20251001
  - ~350 tokens for resource allocation reasoning
  - Serializes top 5 hotspots to structured prompt

Folium Mapping:
  - OpenStreetMap base tiles
  - Circle markers + popups per hotspot
  - Optional heatmap layer with KDE density
  - Exports to HTML for dashboards


PERFORMANCE
================================================================================

1K points       |  <100ms       |  <50ms           |  ~150ms

  - Classification is O(n) linear

  - Folium map generation is O(m) where m = number of hotspots
