NYC DOT SOCRATA TOOLKIT - ACID RELIABILITY FIXES
Quick Reference Summary
==============================================================

AUDIT FINDINGS (4 CRITICAL VIOLATIONS):
1. Atomicity: DuckDB writes sync, L2 Parquet async → inconsistent visibility
2. Consistency: DuckDB↔Parquet skew, concurrent watermark misses data
3. Isolation: Multiple DuckDB connections without pooling, no thread safety
4. Durability: Async L2 writes not durable, session_state never persists

==============================================================
FIX 1: DuckDB CONNECTION POOLING (2 hours)
==============================================================

File: src/socrata_toolkit/core/duckdb_store.py

Change: Add threading.RLock to DuckDBManager

Core modifications:
  - Add self._conn_lock = threading.RLock() in __init__
  - Add execute_atomic(sql, *args) method that holds lock for entire operation
  - All concurrent access goes through single shared connection

Key code:
  def execute_atomic(self, sql: str, *args):
      with self._conn_lock:
          return self.conn.execute(sql, *args)

Testing:
  - Concurrent writes from 5 threads to same table (verify no lost writes)
  - Mixed read/write under thread pool
  - Regression: existing upsert_dataframe() calls still work

Impact: Fixes ISOLATION violation
  ✅ Single connection = implicit transaction context
  ✅ Lock prevents concurrent command execution on same connection
  ✅ No session state conflicts

FIX 2: TRANSACTIONAL WRITES FOR DUCKDB→L2 PARQUET (4 hours)
==============================================================

File: app/utils/cache_manager.py

Change: Wrap DuckDB write + Parquet write + Manifest update in single transaction

Core modifications:
  - Write to .tmp Parquet first
  - BEGIN TRANSACTION
  - Atomic rename .tmp → final
  - Log to DuckDB cache_audit table (watermark)
  - Update manifest.json
  - COMMIT (or ROLLBACK on any failure)

Key code:
  def write_cache_atomic(key: str, df: DataFrame, manager: DuckDBManager):
      with manager.execute_atomic("BEGIN TRANSACTION"):
          df.to_parquet(temp_dest, ...)
          temp_dest.replace(dest)  # atomic at OS level
          manager.execute_atomic("INSERT INTO cache_audit ...")
          update_manifest(...)
          manager.execute_atomic("COMMIT")

Setup: Create cache_audit table on app init
  CREATE TABLE cache_audit (
      id, key, path, rows, fetched_at, expires_at, ttl_hours, written_at
  )

Testing:
  - Success case: all three artifacts created
  - Failure case: Parquet write fails → rollback, temp cleaned up, no manifest update
  - Concurrent writes and reads
  - Regression: read_cache() and eviction still work

Impact: Fixes ATOMICITY and CONSISTENCY violations
  ✅ All-or-nothing: either all 3 updates visible or none
  ✅ No partial state on disk
  ✅ Watermark in DuckDB = single source of truth for concurrent readers

FIX 3: PERSIST SESSION_STATE TO DUCKDB (4.5 hours)
==============================================================

File: app/services/cache_service.py (new)

Change: Create DuckDBSessionStore for persistent session state

Core modifications:
  - Create session_state table in DuckDB
  - Load persisted state on app startup
  - save_key(key, value) and save_all(state_dict) methods
  - Update app/main.py to init_session_persistence()
  - Call store.save_key() after state changes (or register callback)

Key code:
  class DuckDBSessionStore:
      def load_state(self) -> Dict:
          rows = manager.execute_atomic(
              "SELECT key, value FROM session_state WHERE session_id = ?",
              [self.session_id]
          )
          return {k: json.loads(v) for k, v in rows}

      def save_key(self, key: str, value: Any):
          manager.execute_atomic(
              "INSERT INTO session_state (...) VALUES (...)
               ON CONFLICT (session_id, key) DO UPDATE SET ...",
              [self.session_id, key, json.dumps(value), ...]
          )

Setup in app/main.py:
  manager = DuckDBManager()
  store = init_session_persistence(manager, session_id)
  persisted = store.load_state()
  st.session_state.update(persisted)  # merge
  st.session_state._session_store = store

Usage in pages (app/views/studio.py, etc):
  store = st.session_state.get("_session_store")
  st.session_state["cart"] = new_cart
  if store:
      store.save_key("cart", new_cart)

Testing:
  - Load/save round-trip (verify JSON serialization)
  - Concurrent saves to different keys
  - Streamlit lifecycle: load → modify → simulate refresh → verify persisted
  - Backward compatibility: old apps without store still work

Impact: Fixes DURABILITY violation
  ✅ Session state survives page refresh and crashes
  ✅ Multi-page continuity (user data preserved across navigation)
  ✅ User credentials/preferences persist until logout

FIX 4: FILE LOCKING FOR MANIFEST UPDATES (3 hours)
==============================================================

File: app/utils/cache_manager.py

Change: Add advisory file locks around manifest read-modify-write

Core modifications:
  - Platform-specific lock/unlock (fcntl on Unix, msvcrt on Windows)
  - Write to temp file, atomic rename to manifest.json
  - All manifest reads/writes under exclusive lock
  - Timeout on lock waits (don't block forever)

Key code:
  def update_manifest(key: str, path: Path, rows: int, ttl_hours: float):
      manifest, lock_file = cache_manifest_locked()  # acquires lock
      try:
          manifest[key] = {...}
          _save_manifest_locked(manifest, lock_file)  # write under lock
      finally:
          _unlock_manifest(lock_file)  # always release

Lock acquisition:
  # Unix
  import fcntl
  fcntl.flock(file_handle.fileno(), fcntl.LOCK_EX)

  # Windows
  import msvcrt
  msvcrt.locking(file_handle.fileno(), msvcrt.LK_NBLCK, 1)

Atomic write:
  with tempfile.NamedTemporaryFile(...) as f:
      json.dump(manifest, f)
      temp_path = f.name
  Path(temp_path).replace(MANIFEST_PATH)  # atomic at OS level

Testing:
  - Basic lock/unlock (acquire, modify, release)
  - Concurrent updates from 3 threads (verify no lost entries)
  - Atomic rename (simulate interrupt, verify no corruption)
  - Stress test: 100 rapid updates, verify manifest integrity

Impact: Fixes CONSISTENCY violation
  ✅ Read-modify-write atomicity (no lost updates)
  ✅ No TOCTOU (time-of-check-time-of-use) race conditions
  ✅ Manifest is single source of truth for cache state
