# Join Paths Reference — NYC DOT SIM Datasets

Authoritative join paths between core SIM datasets. Use this before writing
multi-table queries. Paths are generated by the relationship graph in `erd_generator.py`.

## Direct Join Paths

### inspection → violations
```sql
-- DuckDB (L2 cache)
SELECT i.objectid, i.borough, i.inspection_date, v.objectid AS violation_id, v.status
FROM read_parquet('data/cache/inspection.parquet') i
LEFT JOIN read_parquet('data/cache/violations.parquet') v
    ON v.inspection_objectid = i.objectid
WHERE i.borough = 'MN'

-- Socrata: fetch separately and merge in pandas
-- inspection_df.merge(violations_df, left_on='objectid', right_on='inspection_objectid', how='left')
```

**Cardinality:** one inspection → zero or many violations
**Relationship confidence:** HIGH (explicit FK in violations schema)

---

### violations → dismissals
```sql
SELECT v.objectid, v.borough, v.status, d.dismiss_reason, d.created_date AS dismiss_date
FROM violations v
LEFT JOIN dismissals d ON d.violation_objectid = v.objectid
```

**Cardinality:** one violation → zero or one dismissal
**Relationship confidence:** HIGH (explicit FK in dismissals schema)

---

### inspection → tree_damage
```sql
SELECT i.objectid, i.borough, t.objectid AS tree_damage_id
FROM inspection i
LEFT JOIN tree_damage t ON t.inspection_objectid = i.objectid
```

**Cardinality:** one inspection → zero or many tree damage records
**Relationship confidence:** HIGH

---

### inspection → violations → dismissals (three-table chain)
```sql
SELECT
    i.objectid      AS inspection_id,
    i.borough,
    i.inspection_date,
    v.objectid      AS violation_id,
    v.violation_type,
    d.dismiss_reason
FROM inspection i
LEFT JOIN violations v ON v.inspection_objectid = i.objectid
LEFT JOIN dismissals d ON d.violation_objectid = v.objectid
WHERE i.borough = 'BK'
  AND i.inspection_date >= '2026-01-01T00:00:00'
```

### inspection ↔ street_permits (spatial join, no FK)
```sql
-- DuckDB spatial join (requires geopandas or DuckDB spatial extension)
SELECT i.objectid, i.borough, p.permit_si_no, p.work_type
FROM inspection i
JOIN street_permits p
    ON ST_DWithin(
        ST_GeomFromGeoJSON(i.the_geom),
        ST_GeomFromGeoJSON(p.the_geom),
        0.0005  -- ~50m in degrees
    )
WHERE i.borough = 'MN'

-- Python alternative using socrata_toolkit
-- from socrata_toolkit.spatial.core import spatial_intersects_join
-- result = spatial_intersects_join(inspection_df, permits_df, 'the_geom', 'the_geom')
```

**Cardinality:** many-to-many (spatial proximity, not FK)
**Relationship confidence:** INFERRED (no FK — join on geometry)

### ramp_complaints ↔ ramp_progress (spatial, no FK)
```sql
-- No shared key — join on geographic proximity or borough + date
SELECT rc.unique_key, rc.borough, rp.objectid AS progress_id, rp.status
FROM ramp_complaints rc
JOIN ramp_progress rp
    ON upper(rc.borough) = upper(rp.borough)
    AND ST_DWithin(
        ST_GeomFromGeoJSON(rc.the_geom),
        ST_GeomFromGeoJSON(rp.the_geom),
        0.001
    )
```

**Relationship confidence:** INFERRED (spatial proximity)

## Common Anti-Patterns

### Joining inspection to violations on borough only
```sql
-- WRONG — produces a Cartesian product within each borough
SELECT * FROM inspection i JOIN violations v ON i.borough = v.borough
```
Always join on `objectid` / `inspection_objectid`. Borough is a filter, not a join key.

### Using objectid across datasets
`objectid` values are not globally unique across datasets — an inspection and a violation
can both have `objectid = 12345`. Never join across datasets using raw `objectid` unless
the FK column (`inspection_objectid`, `violation_objectid`) is used.

## Pandas Join Reference

```python
import pandas as pd

# inspection → violations (one-to-many)
merged = inspection_df.merge(
    violations_df.rename(columns={"objectid": "violation_id"}),
    left_on="objectid",
    right_on="inspection_objectid",
    how="left",
    suffixes=("_insp", "_viol"),
)

# Filter to open violations only
open_violations = merged[merged["status_viol"] == "OPEN"]
```
