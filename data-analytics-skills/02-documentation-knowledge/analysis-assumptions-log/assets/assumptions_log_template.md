### A002 — Null borough treated as UNKNOWN

**Description:** Records where `borough` is NULL or empty string are assigned the label `UNKNOWN` and excluded from per-borough breakdowns but included in city-wide aggregates.
**Rationale:** NULL borough represents <0.2% of the inspection dataset; exclusion from borough charts prevents misleading totals without distorting city aggregates.
**Alternatives considered:** Geocode the associated `unit_id` to derive borough; impute with modal borough for that `unit_id`.
**Impact if wrong:** Low — null rate is consistently <0.2% in inspection (dntt-gqwq).
**Confidence:** High
**Dataset:** inspection  **Column:** borough
**Validation status:** Validated
**Validation notes:** Confirmed null rate 0.18% as of 2026-06-05 via `profile_dataframe()`.

### A003 — Inspection date as event timestamp

**Description:** `inspection_date` is used as the event timestamp for time-series aggregations, not `created_date`.
**Rationale:** `created_date` reflects when the record was entered into the system, not when the inspection occurred. Lag between the two ranges from 0 to 45 days.
**Alternatives considered:** Use `created_date` for operational freshness queries; use `inspection_date` for program analysis.
**Impact if wrong:** Medium — would shift time-series by 0–45 days and inflate recent-period counts.
**Confidence:** High
**Dataset:** inspection  **Column:** inspection_date
**Validation status:** Validated
