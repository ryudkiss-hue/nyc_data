## Potential issues

- **Null borough rows:** Records with null `borough` form their own group (NULL) in the
  output, which can be mistaken for a legitimate borough. Add `WHERE borough IS NOT NULL`
  if NULL groups should be excluded.
- **Division by zero:** `pass_count / total_inspections` will error if `total_inspections`
  is 0 for a borough. Use `NULLIF(total_inspections, 0)` in the denominator.
- **Hardcoded date:** `inspection_date >= '2026-01-01T00:00:00'` — update this when
  re-running for a different year.
- **LEFT JOIN fan-out:** If a unit_id has multiple violation records, the LEFT JOIN
  produces multiple rows per inspection. Aggregate violations before joining, or use
  `COUNT(DISTINCT inspection.objectid)` instead of `COUNT(*)`.

SELECT borough, count(*) AS total_inspections,
       sum(case(status='PASS', 1, true, 0)) AS pass_count
