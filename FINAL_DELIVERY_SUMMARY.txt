Location: src/socrata_toolkit/analysis/hotspot_workflow.py

LangGraph Workflow:
  fetch_data -> spatial_cluster -> classify -> claude_decision ->
  generate_map -> aggregate

Node Functions:
  1. _node_fetch_data: Load violations, complaints, inspections from Socrata
     - Applies WHERE filters for geometry + borough constraints
     - Respects SOCRATA_APP_TOKEN for full-corpus access

  2. _node_spatial_cluster: DBSCAN clustering + KDE density estimation
     - eps=0.0005 degrees (~50m at NYC latitude)
     - min_samples=10 (configurable)
     - Returns cluster centroids and member lists

  3. _node_classify: Apply HotspotClassificationEngine to each cluster
     - Generates hotspots list with severity scores
     - Filters high-severity (>= 50) for Claude reasoning

  4. _node_claude_decision: Invoke Claude API for resource guidance
     - Model: claude-haiku-4-5-20251001
     - ~350 tokens for top 5 hotspots
     - Gracefully skips if API unavailable

  5. _node_generate_map: Create folium visualization
     - Circle markers per hotspot (colored by severity)
     - Popup with full metrics
     - Optional KDE heatmap layer
     - Exports to standalone HTML

  6. _node_aggregate: Build final JSON report
     - Combines all analysis results
     - Includes execution log and error tracking

GeographicHotspotWorkflow Class:
  - Orchestrates complete pipeline
  - Configurable: dbscan_eps, dbscan_min_samples, sample_size, borough_filter
  - Fallback to sequential execution if LangGraph unavailable
  - Returns structured HotspotState with all artifacts

CLASSIFICATION SCORING FORMULA
================================================================================

Severity Score (0-100):
  severity = (density_score * 0.4) + (trend_score * 0.3) + (resource_score * 0.3)

Density Scoring:
  HIGH (>= 50 events/sq km)   = 40 points
  MEDIUM (15-50)               = 20 points
  LOW (< 15)                   = 5 points

Trend Scoring:
  GROWING (> +15% change)      = 30 points
  STABLE (-15% to +15%)        = 15 points
  SHRINKING (< -15%)           = 5 points

Resource Efficiency Scoring:
  UNDER_RESOURCED (< 0.5)      = 30 points
  OPTIMIZED (0.5-0.7)          = 15 points
  OVER_RESOURCED (> 0.7)       = 5 points

Resource Allocation Logic:
  IF density > 50 OR spike > 30% OR (GROWING AND efficiency < 0.7)
     THEN UNDER_RESOURCED
  ELSE IF density < 15 AND SHRINKING AND efficiency > 0.7
     THEN OVER_RESOURCED
  ELSE
     OPTIMIZED

USAGE EXAMPLE
================================================================================

from socrata_toolkit.analysis.hotspot_workflow import GeographicHotspotWorkflow

# Initialize
workflow = GeographicHotspotWorkflow(
    violations_fourfour="dntt-gqwq",
    complaints_fourfour="erm2-nwe9",
    inspections_fourfour="p7ve-f997",
)

# Run
result = workflow.run(borough_filter="MN", sample_size=5000)

# Access results
print(f"Total hotspots: {result['summary']['total_hotspots']}")
print(f"High-severity: {result['summary']['high_severity_count']}")

for hotspot in result['hotspots'][:3]:
    print(f"  {hotspot['hotspot_id']}: "
          f"severity={hotspot['severity_score']:.0f}, "
          f"trend={hotspot['trend']}")

# Save artifacts
with open("map.html", "w") as f:
    f.write(result['map_html'])

import json
with open("report.json", "w") as f:
    json.dump(result['final_report'], f, indent=2)
