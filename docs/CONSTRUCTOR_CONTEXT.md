- **Concurrency**: `ThreadPoolExecutor` (Capped at 3 workers for Socrata to avoid 429 errors).

## 3. Core Logic Summary

### Ingestion (`app/data_loader.py`)
- Centralized fetcher with pre-compiled regex for BBL normalization.
- Parallel dataset loading with network throttling.
- DuckDB synchronous write with background L2 Parquet/Diskcache writes.

### Analytics (`app/analytics.py`)
- **Workflows**: QA/QC Ledger, Spatial Conflict Detection, Contract Dispatch, Productivity Dashboard.
- **Profiling**: Concurrent dataset profiling using schema memoization (MD5 of columns + shape) and LRU caching for geo/date detection.
- **Performance Fixes**: Bypassing date parsing if already in datetime format.

### UI Orchestration (`app/dash_app.py`)
- Single-page application with split-pane forensic layouts.
- Heavy use of pattern-matching callbacks (`ALL`) for dynamic asset population.
- Serverside output caching for high-density DataFrames.

## 4. Performance Hotspots
- **Profiling Latency**: O(N) column scans in `profile_dataset`.
- **BBL Processing**: Regex overhead on 100k+ row batches.
- **Ingestion Failures**: Historical 429 errors from uncapped threading.

## 5. Refactoring Targets
- **God Functions**: `run_all_workflows` (Orchestrator), `profile_dataset` (Multi-responsibility).
- **Coupling**: UI callbacks tightly coupled to dataset logic.
- **Modernization**: Replace `dataclasses` with Pydantic for municipal validation.
