THE 4 FIXES AT A GLANCE:
================================================================================

Fix 1: DuckDB Connection Pooling
  Problem: Multiple connections, no thread safety → ISOLATION violation
  Solution: Single connection + threading.RLock
  File: src/socrata_toolkit/core/duckdb_store.py
  Effort: 2 hours
  Criticality: HIGH (foundation for Fix 2)

Fix 2: Transactional Writes (DuckDB → L2 Parquet)
  Problem: Async writes → ATOMICITY violation
  Solution: BEGIN...COMMIT + atomic rename
  File: app/utils/cache_manager.py
  Effort: 4 hours
  Criticality: HIGH (main data integrity issue)

Fix 3: Session State Persistence
  Problem: session_state lost on refresh → DURABILITY violation
  Solution: DuckDB-backed session store
  File: app/services/cache_service.py (new)
  Effort: 4.5 hours
  Criticality: MEDIUM (user experience)

Fix 4: Manifest Locking
  Problem: Concurrent read-modify-write → CONSISTENCY violation
  Solution: fcntl/msvcrt file locks + atomic rename
  File: app/utils/cache_manager.py
  Effort: 3 hours
  Criticality: HIGH (metadata integrity)

TOTAL EFFORT: ~13.5 hours
