Frequently asked questions about Manhattan Mission Control.


        


          

            
              Q Do I need an account or API key to use this?
              
            
            

              No account needed! All Socrata open data APIs are publicly accessible. You can search, browse, and export datasets without any registration. 


              Optional: Add a free Socrata App Token in the Config panel to increase rate limits from ~1 request/sec to 1000/sec. Get one at opendata.socrata.com.


              The AI Chat feature requires a Google Gemini or OpenAI API key — enter it in Config → API Keys. These are only stored in your browser session, never sent anywhere except the AI provider.
            

          


          

            
              Q How do I save my work and come back to it later?
              
            
            

              There are three ways:


              1. Share URL — Click the Share button in the top-right. Your entire cart and settings are encoded in the URL hash. Bookmark it or send it to a colleague.


              2. Workspaces — Click 📁 Workspaces in the sidebar, give your session a name, and save it. Workspaces are stored in your browser's localStorage and persist across page reloads.


              3. Favorites ★ — Star individual datasets to pin them to your Favorites list, accessible from the sidebar. Favorites also persist in localStorage.
            

          


          

            
              Q What is SOQL and how is it different from SQL?
              
            
            

              SOQL (Socrata Query Language) is Socrata's SQL-like query dialect used to filter and aggregate their datasets via URL parameters.


              Key differences from standard SQL:

              • Uses $where, $select, $limit prefixes

              • No JOIN — joins must be done client-side

              • Supports geospatial functions like within_circle()

              • Max 50,000 rows per query (use $offset to paginate)


              Example: SELECT borough, COUNT(*) WHERE status='Open' GROUP BY borough
            

          


          

            
              Q Why do some map datasets not show up on the map?
              
            
            

              The map only renders datasets that have geographic coordinates. A dataset needs either:


              • A column named location, latitude/longitude, or the_geom

              • A Socrata datatype of Location or Point


              Datasets without geographic data (e.g., contract tables, financial records) will still appear in the cart but won't add any points to the map. Check the dataset's column list in the Profile panel to confirm it has spatial fields.
            

          


          

            
              Q How does the Schema Comparison tool work?
              
            
            

              Click ⚖ Compare Schemas in the sidebar (or the Compare button on the cart). Pick two datasets and the tool produces a side-by-side diff:


              • 🟢 Shared columns — appear in both (potential join keys)

              • 🔴 Unique to A or Unique to B — only in one dataset


              This is especially useful for finding BBL, block_id, or permit_number join keys across NYC datasets before writing a SOQL merge.
            

          


          

            
              Q Is my data sent to any server? Is it private?
              
            
            

              This app is 100% client-side — it's a single HTML file with no backend. Nothing you do is sent to our servers because there are no servers.


              Data flows:

              • Socrata API — your browser queries Socrata directly (public open data)

              • AI Chat — your message + dataset metadata goes to Google/OpenAI (only if you enter an API key)

              • localStorage — workspaces and favorites are stored only in your browser

              • URL hash — cart state is in the URL, visible only to people you share it with
            

          


          

            
              Q How do I download the generated diagram as a file?
              
            
            

              After generating a diagram in the Diagram tab:


              • Click ⬇ Download SVG — saves a vector SVG (scalable to any size, editable in Figma/Illustrator)

              • Right-click the diagram → "Save image as…" — for PNG


              The SVG download preserves all styling and is suitable for presentations, documentation, or data dictionaries.
            

          


          

            
              Q Can I use this app offline?
              
            
            

              Partially. The HTML file itself works offline, but most features need internet access:


              Works offline: The UI, saved workspaces, favorites, code generator (from cart metadata already loaded)


              Needs internet: Search (hits Socrata API), Map tiles (CDN), AI Chat (AI provider), Diagram rendering (Mermaid CDN), column profiling (fetches live data)


              Tip: If you've already searched and have results in your cart, the Code Generator and Diagram features will work offline since they use cart metadata, not live data.

Power-user tips to get the most out of Mission Control.


        


          

            
🔗

            

              
Find Join Keys Between Datasets

              
Add two NYC datasets to your cart, then use ⚖ Compare Schemas. Green-highlighted columns appear in both datasets — these are your join keys. Common ones: bbl, bin, block, permit_number, unique_key.

            

          


          

            
📋

            

              
Multi-City Research in One Session

              
Enable Search All Cities in the sidebar. Then search for a common dataset type like "permits" — you'll get results from NYC, Chicago, LA, and more in a single result list, perfect for cross-city policy analysis.

            

          


          

            
⚡

            

              
Generate a Full dbt Project in 30 Seconds

              
Search for 3–4 related datasets (e.g., inspections, violations, permits), add them all to your cart, go to Code tab, select dbt Models, and click Generate. You'll get a complete dbt project with sources, staging models, and a mart layer.

            

          


          

            
🎯

            

              
Use SOQL for Targeted Data Pulls

              
Instead of downloading an entire 1M-row dataset, use the SOQL runner to pull only what you need. Example: SELECT * WHERE borough='MANHATTAN' AND created_date > '2024-01-01' LIMIT 5000. Then export just those rows as CSV.

            

          


          

            
🤖

            

              
Let AI Write Your SOQL Queries

              
Add datasets to your cart, then go to AI Chat and ask: "Write a SOQL query to find all open 311 complaints in Manhattan from the last 30 days, grouped by complaint type." The AI has your cart context and produces ready-to-run queries.

            

          


          

            
🗺️

            

              
Layer Multiple Datasets on One Map

              
Add several geo datasets to your cart (e.g., 311 complaints + sidewalk inspections + tree census), then click 🗺 Full Map View. All datasets render as separate layer groups you can toggle on/off individually.

            

          


          

            
💾

            

              
Share Research with Your Team

              
After building your cart, click Share in the top-right header. The URL encodes your entire workspace. Anyone who opens the link sees your exact cart, search state, and domain selection — no login required.

            

          


          

            
📐

            

              
Use Mindmaps for Dataset Discovery

              
Start with one anchor dataset (e.g., "Street Permits"), go to Diagram → Mindmap, and generate. The mindmap shows the dataset's columns as branches. Use it to spot categories, then search for datasets that complement each branch.

            

          


        

      


      
      

        
Key terms used throughout NYC open data and this application.


        

          

            ID
            

              
BBL — Borough-Block-Lot

              
A 10-digit unique identifier for every parcel of land in NYC. Format: B-BBBBB-LLLL (1 digit borough + 5 digit block + 4 digit lot). Used as the primary join key across property, violation, inspection, and tax datasets. Example: 1000477501 = Manhattan, Block 477, Lot 7501.

            

          

        


        

          

            ID
            

              
BIN — Building Identification Number

              
A 7-digit identifier assigned to every physical building structure in NYC by the Department of Buildings (DOB). Unlike BBL which identifies a tax lot, BIN identifies a building — one BBL can have multiple BINs. Used extensively in DOB permits, complaints, and inspection data.

            

          

        


        

          

            API
            

              
SOQL — Socrata Query Language

              
A SQL-like query language used to filter and aggregate Socrata open data via REST API. Queries are URL parameters appended to the dataset endpoint. Key clauses: $select, $where, $group, $order, $limit, $offset. Max 50k rows per request — use $offset to paginate.

            

          

        


        

          

            ID
            

              
Fourfour ID (4×4)

              
The unique 8-character identifier Socrata assigns to every dataset. Format: xxxx-xxxx (4 alphanumeric characters, a dash, then 4 more). Example: erm2-nwe9 is NYC's 311 dataset. Appears in every dataset URL and API endpoint. This app displays the fourfour ID on each cart item.

            

          

        


        

          

            Auth
            

              
Socrata App Token

              
A free API credential that increases your Socrata rate limit from ~1 req/sec (throttled) to 1,000 req/sec (generous). Register at opendata.socrata.com. Enter it in Config → API Keys. Without it the app still works, just slower on large result sets.

            

          

        


        

          

            Geo
            

              
the_geom / Point / MultiPolygon

              
Socrata stores geographic data in a column commonly named the_geom using GeoJSON or WKT format. Point = a single lat/lon coordinate. MultiPolygon = a complex shape like a borough boundary. This app auto-detects geometry columns to render markers and heatmaps on the map.

            

          

        


        

          

            NYC
            

              
MapPLUTO

              
NYC Department of City Planning's comprehensive land use and geographic dataset. Contains tax lot info (BBL, address, zoning, building class, year built, floor area, owner name) for every parcel in the city. The definitive reference dataset for NYC property analysis.

            

          

        


        

          

            NYC
            

              
SIM — Sidewalk Inspection Management

              
NYC DOT's program for managing and tracking sidewalk repair orders throughout Manhattan. SIM data includes inspection outcomes, repair assignments, contractor tracking, and payment milestones. The core operational dataset driving this application's analytics.

            

          

        


        

          

            Data
            

              
Parquet Cache

              
In the Streamlit backend, datasets fetched from Socrata are stored locally as Apache Parquet files (columnar binary format) to avoid re-fetching on every page load. Caches expire after 24 hours. Manage them in Streamlit → Settings → Cache tab.

            

          

        


        

          

            Data
            

              
dbt — Data Build Tool

              
An open-source SQL transformation framework. This app's Code Generator can produce dbt source YAML (declaring Socrata datasets as raw sources), staging models (type casting + renaming), and a mart model (joining datasets). dbt models are version-controlled SQL files that compile into your warehouse.

            

          

        


        

          

            Quality
            

              
Great Expectations

              
An open-source data quality framework. The Code Generator produces GE Expectation Suites — YAML/JSON files that define rules like "column X should never be null", "column Y should be between 0 and 100", "column Z should match regex". Run suites against live data to catch pipeline issues early.

            

          

        


        

          

            NYC
            

              
Borough Codes

              
NYC uses single-digit codes in BBL and many datasets: 1 = Manhattan, 2 = Bronx, 3 = Brooklyn, 4 = Queens, 5 = Staten Island. Some datasets use the text names (MANHATTAN) while others use the numeric codes — always check before joining.
