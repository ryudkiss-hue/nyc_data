# 🗽 Product Guidelines - SIM Analyst Mission Control

## 1. Visual Identity & Branding
*   **Aesthetic**: Modern Corporate. The application should maintain a clean, professional appearance suitable for executive reviews and daily municipal operations.
*   **Palette**: Utilize a crisp, high-contrast palette that supports clear data visualization and legible typography.
*   **UI Framework**: Implement components with high information density to maximize screen real estate for complex data analysis, while ensuring polished, interactive feedback for user actions.

## 2. Prose & Communication Style
*   **Voice**: Executive Professional and Technical Forensic. Content must be direct, data-led, and use precise municipal nomenclature (e.g., JIDs, Agency Codes).
*   **Accessibility**: All insights must support a toggle or default to an 8th-Grade Reading Level to support cross-departmental understanding.
*   **Acronym Expansion**: All technical acronyms (SODA, OMB, DOT, JID) must be expanded or defined upon first use to ensure clarity.
*   **Tone**: Avoid conversational filler; prioritize high-signal delivery of facts and findings.

## 3. Data Presentation Standards
*   **Quantitative Integrity**: Maintain a "Zero Mocking" policy. Use explicit "N/A" or "Data Unavailable" for missing values; never substitute `0` for missing data.
*   **Statistical Reporting**: All analytical summaries must include the "Four Moments of Data Quality":
    1.  Expected Value (Mean)
    2.  Variance (Volatility)
    3.  Skewness (Asymmetry)
    4.  Kurtosis (Tail Risk)
*   **Reliability Tagging**: Every visualization and dataset must display a prominent Reliability/Completeness score based on source record counts and data freshness.

## 4. Technical UI Principles
*   **Density**: Prioritize layouts that allow for simultaneous viewing of maps, charts, and data grids (e.g., Split-pane Forensic Audits).
*   **Feedback**: Provide immediate visual confirmation for all background data loads, exports, and agential queries.
