1Invalid SoQL Date Syntax in Example

HIGH — Breaking Example

Location: CLAUDE.md, line 539

The Problem

$where=upper(borough)='MANHATTAN' AND created_date>'<30d ago>'&$select=count(*)
  • Relative date syntax <30d ago> is not valid SoQL
  • Socrata SODA2 API does not support relative date operators
  • Mixing & (URL separator) into WHERE clause is incorrect
  • $select=count(*) without aggregation syntax is invalid

Impact

Users copy-pasting this example will receive:

400 Bad Request: Query contains syntax error

Verification

✓ CONFIRMED — Actual client code uses ISO 8601 timestamps like '2026-05-06T00:00:00'

Recommended Fix

Replace with valid ISO 8601 timestamp:

created_date > '2026-05-06T00:00:00'&$limit=50000

Or show dynamic date computation:

from datetime import datetime, timedelta
thirty_days_ago = (datetime.now() - timedelta(days=30)).isoformat()
# Then: $where=upper(borough)='MANHATTAN' AND created_date > '{thirty_days_ago}'
